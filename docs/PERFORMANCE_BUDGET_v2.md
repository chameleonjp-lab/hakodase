# HAKODASE v2 性能契約

## 基本方針
HAKODASEは「広域対応型」とする。共通の3MB上限、共通の総メモリ上限、共通の物体数上限は設けず、HAKODASE固有の速度、負荷、継続試験で判断する。

## 目標
```text
60Hz端末で滑らかな60fpsを目標
高更新端末でもアニメ時間を変えない
入力後50ms以内の視覚反応を目標
プレイ中の50ms超の長い処理は調査対象
100ms超の意図しない停止は公開停止条件
```

「60fps固定」とは扱わない。ブラウザの表示更新に合わせ、経過時間で動きを計算する。

## 測定対象
- 最も重い盤面。
- 最大箱数。
- 最大壁数。
- 最大ギミック数。
- 最大パーティクル数。
- 連続再挑戦10回。
- 30分連続プレイ。
- バックグラウンド移動と復帰10回。
- 通信切断と再接続。
- 周回ごとにイベント、粒子、配列、描画資源が増え続けないこと。
- 通常より負荷を20％増やした試験。
- 同じ公開候補版で3回連続合格。

## 最も重い場面候補
主盤面7×9、箱14個、色6色、壁と出荷レーンと出荷シャッターが最大級、粒子が発生し、結果表示または送信状態表示が重なる場面を候補にする。具体値はPhase 6で測定用に確定する。

## 長い処理への扱い
プレイ中に50ms超の処理が見つかった場合は調査対象にする。100ms超の意図しない停止が公開候補版で発生する場合は公開停止条件にする。

## 実機未確認
今回の文書作業では実機測定を行っていない。最低保証候補端末とブラウザの合否は未確認であり、Phase 6で測定する。
