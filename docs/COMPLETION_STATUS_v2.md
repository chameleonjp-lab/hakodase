## P2-03で実装したこと

### カウントダウン

```text
3 → 2 → 1 → START
```

- 既定900ミリ秒間隔。
- STARTまで盤面を表示しない。
- 開始ごとに世代tokenを発行する。
- 中止、再挑戦、新規盤面後の古いタイマーを無効にする。
- カウントダウン中にホームへ戻れる。

### 時計の開始

- START時の同じ描画フレームで`countdown → playing`へ遷移する。
- 最初の盤面を描画する。
- 同じ`frameTime`を`GameEngine.start()`と`RunController.start()`へ渡す。
- 両方が成功した場合だけ入力を許可する。
- 二重START、古い`playId`、不正な時刻を拒否する。
- 一方だけ開始に失敗した場合はプレイを開始しない。

### 再挑戦

- `playing → countdown`を許可した。
- やりなおし、新しい盤面、seed指定は新しい`playId`を発行する。
- 再開始も必ずカウントダウンを通る。

### 厳格時計

- 本日の出荷だけ`strictClock: true`。
- カウントダウン中またはプレイ中にページが隠れた場合、試行を無効にしてホームへ戻す。
- P2-03時点の本日の出荷は暫定問題かつ記録対象外。
- エンドレスと練習は画面非表示だけでは無効にしない。一時停止は未実装。

### 構造

```text
src/app/countdown-controller.js
src/app/start-run.js
src/app/visibility-policy.js
src/ui/countdown-flow.js
src/p2-03-bootstrap.js
```

純粋制御とDOM統合を分け、P2-02までの`pendingStart`開始を実行時に無効化した。

詳細は[P2-03設計](P2_03_COUNTDOWN_CLOCK_v2.md)を参照する。
