## P2-05で実装したこと

### 結果の一回表示

- 現在の`playId`が`cleared`の場合だけ結果画面へ進む。
- 処理中、表示待ち、表示済みを分けて二重遷移を防ぐ。
- 古い`playId`の遅延結果は破棄する。
- クリア時に固定済みの時間を表示し、演出待ち時間を加算しない。
- リタイアと無効化は結果記録を作らない。

### 結果指標

```text
クリアタイム
操作数
移動マス
undo回数
最短または目安
最短との差
モード
問題IDまたはseed
```

旧MVP盤面で厳密性が確認できない値は「目安」と表示する。

### 端末内初回・ベスト

- `mode + difficulty + seed`が同じ記録だけを集計する。
- 初回は最古の達成日時。
- ベストはタイム、操作数、達成日時の順。
- エンドレスだけを対象とする。
- 練習と本日の出荷プレビューは記録対象外と表示する。

### 再挑戦と次の行動

- 同じseedで新しい`playId`を発行し、`result → countdown`へ進む。
- 再挑戦も3・2・1・STARTを通る。
- エンドレスだけ新しい盤面へ進める。
- ホームへ戻ると盤面と現在プレイ参照を破棄する。

### 共有

```text
Web Share
→ 技術的失敗時 Clipboard
→ 失敗時 選択可能テキスト
```

- 利用者キャンセルはエラー扱いにしない。
- 共有不能でも結果画面を維持する。
- 現在のページURLを使用し、未確定の公開URLを固定しない。

### 構造

```text

src/services/share-result.js
src/ui/result-flow.js
src/p2-05-bootstrap.js
styles/p2-05.css
```

詳細は[P2-05設計](P2_05_RESULT_SHARE_v2.md)を参照する。
