# P3-02 厳密ソルバーv2 決定記録

日付: 2026-07-22

## 決定

1. 1スライドを1コストとする深さ順の幅優先探索を厳密最短の正本とする。
2. `slide-exit/1`では同色箱のID交換がゲーム規則へ影響しないため、状態keyだけを色ごとの位置集合へ正規化する。
3. 探索queueには実際の箱ID配置を保持し、解法列は`blockId`と開始位置を含めて再生可能にする。
4. 箱順はP3-01正規化後のID順、方向順は`up, down, left, right`へ固定する。
5. 残存箱数と搬出口の行・列一致から得る下界だけを、最大深さの安全な枝刈りに使用する。
6. ノード、状態数、時間、深さ、中断の上限へ到達した場合、`optimalSwipes`は`null`とし推定値を保存しない。
7. 盤面を完全探索して解がない場合だけ`exact: true / reason: exhausted`とする。
8. 出荷シャッターは未対応として探索前に拒否し、黙って壁や空きセルとして扱わない。
9. ソルバーは開発時・候補検査時専用とし、公開ゲームの開始処理や描画ループから呼ばない。
10. 8箱、11箱、14箱の容量fixtureは正しさと性能の検査専用とし、公式問題や面白さの証拠には使用しない。

## 理由

- 同色箱を個別IDのまま探索すると、同じ盤面状態を箱IDの順列だけで大量に重複探索する。
- 同色箱の個別能力がない現行ルールでは、位置集合が同じなら将来の合法手と最短残り操作数も同じである。
- A*などの強い推定へ進む前に、正しさを説明しやすい厳密BFSと安全な下界を基準として持つ必要がある。
- 候補生成の都合で上限到達値を最短値として扱うと、公式問題の表示とランキング条件が破綻する。
- 8〜14箱のすべての盤面を固定予算内で解ける保証はないため、P3-03は解けない候補を理由付きで不採用にする必要がある。

## 将来変更条件

次の場合は対称性圧縮の再検討と`rulesVersion`更新が必要になる。

- 同色箱ごとに重量、形、能力、移動制限が異なる。
- 箱IDがシャッターやレーンの判定へ影響する。
- 1×1以外の形状を導入する。
- 時間または手数位相を状態へ含める。
