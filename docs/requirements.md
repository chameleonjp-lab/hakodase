# HAKODASE / ハコダセ 要件仕様書

短時間タイムアタック型の「色ブロック搬出パズル」。ブラウザだけで動く、ビルド不要の静的サイト。

> 「Block Out! - Color Sort Puzzle」にインスパイアされた**独自実装**。名称・画面・面データ・素材・文章は流用しない（§15）。

---

## 1. ゲーム概要

- ジャンル: ブラウザ向けの短時間タイムアタック型スライド搬出パズル。
- テーマ: **倉庫の色分け出荷**。色のついた「コンテナ（箱）」を、同じ色の「搬出口（出口ゲート）」から盤外へ出荷する。
- 中核: 箱をドラッグすると **壁・他の箱・盤端に当たるまで一気に滑る**。各箱を **自分と同じ色の出口ゲートから盤外へ退場** させ、すべて出すとクリア。
- 出す順序が要（先に動かすと他を塞ぐ）。クリアタイムを競う。
- 盤面はランダム生成。ランキング対象は「最短20手以上」の盤面のみ採用する。

## 2. 対象端末

- **第一優先: iPhone SE 相当（横幅 320px 程度）のスマートフォン縦持ち。** 盤面は縦長 7×9 を基本。
- PC ブラウザ（マウス）でも遊べる。Pointer Events でタッチ/マウス統一。
- モダンブラウザ（Pointer Events / Canvas2D / localStorage / ES Modules）。

## 3. MVP の範囲（必ず入れる）

| 項目 | 内容 |
| --- | --- |
| 盤面 | グリッド（練習 5×6〜ランキング 7×9） |
| ブロック | 1×1 のコンテナ（データ構造は w/h を持つ） |
| 移動 | ドラッグ方向へ壁・他ブロック・盤端まで一気にスライド |
| 退場 | 同色の出口ゲート（盤の縁）から盤外へ出すと達成 |
| クリア | 全ブロック退場 |
| 障害物 | 壁（スライドを止める） |
| 色数 | 2〜6 色（記号併記で色覚配慮） |
| 生成 | seed 付き・再現可能。逆生成で常に可解。ランキングは最短20手以上 |
| タイムアタック | クリアタイムを ms で記録、`12.34秒` 表示 |
| ローカルランキング | localStorage（オンラインは将来差し替え可能な interface） |
| 操作 | リトライ・新規盤面・難易度選択・seed 指定・ランキング表示 |
| 演出 | 退場時/クリア時パーティクル、選択ハイライト、着地プレビュー（すべて Canvas） |

## 4. 後回しにする範囲（MVP 外）

- 1×2 / 2×1 ブロック（データ構造だけ用意）。
- Three.js / WebGL 描画（Renderer interface だけ用意）。
- オンラインランキング（RankingService interface だけ用意）。
- アンドゥ（本家にはあるが、タイム公平性のため MVP は「やりなおし」で代替）。
- 追加ギミック: 一方通行床（ルールは実装済みだが生成では未使用＝後述）、可動ゲート、エレベーター、ブロック生成器。
- 効果音（任意。入れる場合も Web Audio 生成）。

## 5. 基本ルール

- 盤面はグリッド。各セルは空き or 壁。盤の縁に色つき出口ゲートがある。
- ドラッグした方向へ、対象ブロックは進めなくなるまで（壁・他ブロック・盤端）1マスずつ連続して滑る。
- 滑った先が盤端で、その縁に同色の出口ゲートがあれば盤外へ退場（達成）。色が違う/ゲートが無ければ盤端で停止。
- すべてのブロックが退場するとクリア。
- タイマーは **最初の操作で開始**、クリアで停止。やりなおし/新規でリセット。
- 1 手 = 1 スライドで通過したマス数（退場の1歩を含む）。手数の数え方はソルバーと一致（§10）。

## 6. ブロック（コンテナ）仕様

- MVP は 1×1。`{ id, x, y, w, h, color }`（w/h は将来拡張用）。
- 描画は画像なしの Canvas 角丸矩形。面取り・影・グラデ・ハイライト・四隅リベット・エンボス記号。
- 各色に固有の記号（●▲■◆★＋）をブロックと出口ゲートの両方に描く。
- 論理座標は整数グリッド。アニメは見た目のみ。停止時は整数セル、退場時はゲート開口へスナップ。

## 7. 出口ゲート（搬出口）仕様

- 盤の縁に位置する: `{ side: 'left'|'right'|'top'|'bottom', line, color }`。
  - left/right の `line` は行(y)、top/bottom の `line` は列(x)。
  - 盤外の「開口セル」: left=(-1,line) / right=(width,line) / top=(line,-1) / bottom=(line,height)。
- ブロックがその縁に達したとき、**同色のゲートがあれば退場**、無ければ停止。
- 1 色につき 1 ゲート（MVP）。同一縁セルに複数ゲートは置かない。

## 8. 障害物（壁）仕様

- 静的な通行不可セル。スライドはその手前で止まる。
- 壁が箱どうしの「塞ぎ合い」を生み、出す順序の読みを要求する。

## 9. ランダム生成仕様

- seed 付き。**同じ seed なら同じ盤面**。
- **逆生成**: 各ブロックを自分の出口ゲートから盤内へ滑り込ませて配置する。挿入時に通路が空である限り、その逆順が必ず成立する正解手順になる＝**常に可解**。探索不要で高速。
- 各ブロックは自分のゲートと一直線に並ぶため、**最短手数 = マンハッタン距離総和**（解析的に厳密）。
- min20 難易度はマンハッタン距離総和 ≥ 22 を満たす配置のみ採用（§10 の保証）。
- 仕上げに `quickSolvable`（十分条件）で可解性を再確認。
- 配置失敗時は seed を変えて再試行（**最大試行回数あり・無限ループ禁止**）。それでも失敗時は **検証済みフォールバック盤面**。
- フォールバックは各色が独立レーンを真横に滑って出るだけの構成で、常に可解・最短 = width×colors（テストで20手以上を確認）。

## 10. 最低20手の定義

- 「20手」= 論理上の 1 マス移動を 1 手として、1スライドで通過したマス数（退場の1歩含む）の合計。
- ランキング対象の条件: 各ブロック→同色ゲート開口のマンハッタン距離総和（最短手数の下界）≥ 20。
  - 各移動はどれか1ブロックを1マス進めるので、総和は最短手数の下界。解ければ最短手数は必ずその値以上。

## 11. タイムアタック仕様

- クリアタイムを **ミリ秒（整数）** で保持、`12.34秒` 表示。
- タイマーは最初の操作で開始、クリアで停止。やりなおし/新規でリセット。手数も表示。

## 12. ローカルランキング仕様

- localStorage に保存。MVP はオンライン非対応。
- 保存項目: `seed` / `difficulty` / `timeMs` / `moves` / `clearedAt`。
- 並び: タイム昇順 →（同値なら）手数 →（同値なら）達成時刻。難易度でフィルタ表示。
- 将来オンライン化できるよう `RankingService` interface（save/list/clear）で分離。

## 13. 描画・演出仕様

- すべて Canvas2D、外部画像・音声なし。
- コンテナ: 角丸矩形＋面取り＋影＋グラデ＋ハイライト＋リベット＋色記号。
- 出口ゲート: 盤の縁に色バー＋外向き矢印＋色（搬出口を示す）。
- 着地プレビュー: ドラッグ中、滑った先（または退場方向）を破線/矢印で予告。
- 選択中コンテナは明るく＋白リング。退場時・クリア時に Canvas パーティクル。
- devicePixelRatio 対応。requestAnimationFrame。

## 14. パフォーマンス要件

- 60fps 目標。requestAnimationFrame。毎フレームの重い探索・DOM 更新・レイアウト再計算を避ける。
- 生成は逆生成で探索不要・高速（数 ms）。重い処理は将来 Web Worker に移せる純粋関数構造を保つ。
- Canvas は devicePixelRatio 対応。

## 15. オリジナリティ方針

- 既存ゲームをコピーしない。**名称・画面構成・面データ・演出・素材・文章を流用しない。**
- 着想元（Block Out! 系の「色分け搬出スライド」）の**遊びの骨格**のみを参考にし、テーマ・用語・絵・生成・UI はすべて独自に構築。
- テーマは「倉庫の色分け出荷／箱を搬出口へ」。用語は独自語彙（コンテナ・搬出口・出荷）。
- 外部画像・外部音声・既存ゲーム名・似すぎた UI 文言を使わない。視覚は Canvas 自前描画のみ。

## 16. 受け入れ条件

機能:
- ブラウザでゲーム画面が開く。
- コンテナをタッチ／マウスでドラッグし、壁・他ブロック・盤端まで滑らせられる。
- 同色の出口ゲートから盤外へ出すと達成、全部出すとクリア。
- 壁が通行制限として働く。
- 2〜6 色の難易度を選べる。
- ランキング対象では最短20手以上の盤面を出す。
- クリアタイム表示・ローカルランキング記録・やりなおし/新規/ seed 指定ができる。

見た目:
- コンテナは角丸矩形、影・ハイライト/グラデあり。出口ゲートが色で分かる。
- 退場/クリアでパーティクル。小画面（7×9 縦長）でも盤面が見切れない。

設計:
- GameEngine が Canvas に依存しない。Renderer interface がある。
- CanvasRenderer を差し替えれば Three.js/WebGL 版を追加できる。
- RankingService が分離。Solver と Generator が分離。

テスト:
- `npm test` でテストが走る。RNG の再現性。
- スライド/退場判定・壁での停止をテスト。
- ソルバーが最短手数（通過マス数）を返す／可解性チェック。
- ランキング用生成盤面が20手以上・可解であることをテスト。
- RankingService の保存と読み出しをテスト。
