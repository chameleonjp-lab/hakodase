# HAKODASE / ハコダセ 要件仕様書

短時間タイムアタック型の「色ブロック整理パズル」。ブラウザだけで動く、ビルド不要の静的サイト。

> 本書は product-spec-agent / originality-risk-agent / game-engine-agent の観点を統合した要件定義です。

---

## 1. ゲーム概要

- ジャンル: ブラウザ向けの短時間タイムアタック型スライド整理パズル。
- テーマ: **倉庫の色分け出荷**。色のついた「コンテナ（箱）」を、同じ色の「搬出口（ゴール）」へ運んで出荷する。
- 目標: すべての必要コンテナを同じ色の搬出口に入れるとクリア。クリアまでのタイムを競う。
- 1プレイ: 30〜90 秒程度を想定した短時間勝負。
- 盤面はランダム生成が中心。ランキング対象は「最短20手以上」の盤面のみ採用する。

## 2. 対象端末

- **第一優先: iPhone SE 相当（横幅 320px 程度）のスマートフォン縦持ち。**
- PC ブラウザ（マウス操作）でも遊べる。
- モダンブラウザ（Pointer Events / Canvas2D / localStorage / ES Modules 対応）。
- タッチ・マウスの両対応（Pointer Events で統一）。

## 3. MVP の範囲（必ず入れる）

| 項目 | 内容 |
| --- | --- |
| 盤面 | 5×5〜6×6 のグリッド |
| ブロック | 1×1 のコンテナ（データ構造は width/height を持つ） |
| 移動 | 上下左右に 1 マスずつ。ドラッグ方向へ動かし、離すとスナップ |
| ゴール判定 | 同色の搬出口に全コンテナが入るとクリア |
| 一方通行床 | 矢印床。出る方向を矢印方向に制限 |
| 開閉ゲート | 一定手数ごとに開閉。閉時は通行不可 |
| 色数 | 2〜6 色（記号併記で色覚に配慮） |
| ランダム生成 | seed 付き・再現可能。ランキング対象は最短20手以上を保証 |
| ソルバー | 幅優先探索で最短手数を算出（上限つき） |
| タイムアタック | クリアタイムを ms で記録、`12.34秒` 表示 |
| ローカルランキング | localStorage に保存（オンラインは将来差し替え可能な interface） |
| 操作 | リトライ・新規盤面生成・難易度選択・ランキング表示 |
| 演出 | クリア時パーティクル、選択中ハイライト、ゴール点灯（すべて Canvas 描画） |

## 4. 後回しにする範囲（MVP 外）

- 1×2 / 2×1 など複数マスブロック（データ構造だけ先に用意）。
- Three.js / WebGL 描画（差し替え可能な構造だけ用意）。
- オンラインランキング（RankingService interface だけ用意）。
- Web Worker での生成・探索（後で移せる構造だけ用意）。
- 効果音（任意。入れる場合も Web Audio 生成、外部ファイル不使用）。
- ドラッグ1回での複数マス連続移動（内部の手数定義だけ通過マス数で先に定義）。

## 5. 基本ルール

- 盤面はグリッド。各セルには地形（空き / 壁 / 一方通行床 / ゲート / 搬出口）がある。
- プレイヤーは色つきコンテナを上下左右に 1 マスずつ動かす。
- コンテナが同じ色の搬出口セルに乗ると「出荷済み（達成状態）」になる。
- すべての必要コンテナが同色搬出口に入るとクリア。
- タイマーは **最初の操作（最初の1手）で開始**する。
- リトライ時はタイマー・手数を 0 から測り直す。
- 1 手 = 論理上の 1 マス移動（定義は §10）。

## 6. ブロック（コンテナ）仕様

- MVP は 1×1。データ構造は `{ id, x, y, w, h, color }`（w/h は将来拡張用、MVP は 1）。
- 描画は画像を使わず Canvas の角丸矩形。影・ハイライト・グラデーションを付ける。
- 色だけに頼らず、各色に固有の **記号**（●▲■◆★＋ など）をコンテナと搬出口の両方に描く。
- 論理座標は必ず整数グリッド。アニメーションは見た目だけで、停止時は整数セルに正確にスナップ。

## 7. 一方通行床仕様

- 矢印つき床として表示する（上下左右いずれかの向き）。
- **コンテナがその床のセルにいる場合、次に出られる方向は矢印方向のみ**に制限される。
- 戻れないルートや遠回りを作り、最短手数を伸ばす手段になる。
- ソルバーも同じ制約で探索する（ロジックとソルバーで完全一致）。

## 8. 可動ゲート（開閉ゲート）仕様

- セル上のゲート。一定手数ごとに開閉する。
- 閉じていると通れない（そのセルへ進入できない）。開いていると通れる。
- 状態は手数のみで決まる純粋関数: `open(moveCount) = ((moveCount + phase) % period) < openFor`。
- 見た目はスライドするシャッター/バーとして Canvas で描く。
- ソルバーは状態に `moveCount mod cycle`（全ゲート周期の最小公倍数）を含めて探索する。

## 9. ランダム生成仕様

- seed 付き。**同じ seed なら同じ盤面**。
- 難易度に応じて色数・盤面サイズ・ギミック量を調整する。
- ランキング対象盤面は、生成時に「最短20手以上」を保証する。
  - 保証手段: **各コンテナ→同色搬出口のマンハッタン距離の総和を下界**として使い、これが閾値（22）以上になるよう配置する。マンハッタン距離総和は最短手数の下界なので、解けるなら最短手数は必ずその値以上になる。
  - ソルバーで解の存在を確認し、最短手数を厳密に算出して表示・ランキングに使う。
- 生成失敗（解けない・探索上限超過）時は seed を変えて再試行する（**最大試行回数あり、無限ループ禁止**）。
- それでも失敗する場合は、**検証済みフォールバック盤面**を返す（エラーで止めない）。
- フォールバック盤面もテストで「20手以上・解ける」ことを確認する。

## 10. 最低20手の定義

- 「20手」は **論理上の 1 マス移動を 1 手** として数える。
- ドラッグ1回で複数マス動かす設計にしても、内部では通過したマス数を手数として数える（MVP は1ドラッグ=1マス）。
- ランキング対象の条件: `solver.shortestSolutionMoves >= 20`。

## 11. タイムアタック仕様

- クリアタイムを **ミリ秒（整数）** で保持。
- 表示は `12.34秒`（小数第2位まで）。
- タイマーは最初の1手で開始、クリアで停止。リトライ・新規生成でリセット。
- 手数も記録・表示する。

## 12. ローカルランキング仕様

- localStorage に保存する。MVP はオンライン非対応。
- 保存項目: `seed` / `difficulty` / `timeMs` / `moves` / `clearedAt`(ISO文字列)。
- 並びはタイム昇順。難易度でフィルタ表示できる。
- 将来オンライン化できるよう `RankingService` interface（save/list/clear）で分離する。

## 13. 描画・演出仕様

- すべて Canvas2D 描画、外部画像・外部音声を使わない。
- コンテナ: 角丸矩形＋影＋グラデーション＋ハイライト＋色記号。
- 搬出口: 色枠＋色記号。対応コンテナが入ると軽く光る。
- 一方通行床: 矢印。ゲート: スライドするシャッター表現。
- 選択中コンテナは少し明るくする。
- クリア時に Canvas パーティクルを出す。
- devicePixelRatio 対応。requestAnimationFrame で描画。

## 14. パフォーマンス要件

- 60fps を目標。requestAnimationFrame 使用。
- 毎フレームで重い探索・DOM 更新・CSS レイアウト再計算を起こさない。
- 生成・ソルバーは探索上限つき。重くなる場合は将来 Web Worker に移せる構造にする。
- Canvas は devicePixelRatio に合わせる。

## 15. オリジナリティ方針

- 既存ゲームをコピーしない。**名称・画面構成・ステージ・演出・素材・文章を流用しない。**
- 「色ブロックを整理する短時間パズル」という抽象的発想からオリジナルに構築する。
- テーマは「倉庫の色分け出荷／箱を出荷口へ運ぶ」に寄せる。
- 用語は独自語彙（コンテナ・搬出口・搬入・出荷・周回ゲート など）を使う。
- 外部画像・外部音声・既存ゲーム名・似すぎた UI 文言を使わない。
- 視覚は Canvas 自前描画のみで構成し、第三者素材を一切含めない。

## 16. 受け入れ条件

機能:
- ブラウザでゲーム画面が開く。
- コンテナをタッチ／マウスで動かせる。
- 同色搬出口へすべて入れるとクリアになる。
- 一方通行床が移動制限として働く。
- 開閉ゲートが通行制限として働く。
- 2〜6 色の難易度を選べる。
- ランキング対象では最短20手以上の盤面を出す。
- クリアタイムが表示される。
- ローカルランキングに記録される。
- リトライ・新規盤面生成ができる。

見た目:
- コンテナは角丸矩形、影あり、ハイライト/グラデーションあり。
- クリア時にパーティクルが出る。
- 小画面でも盤面が見切れない。

設計:
- GameEngine が Canvas に依存しない。
- Renderer interface がある。
- CanvasRenderer を差し替えれば Three.js/WebGL 版を追加できる。
- RankingService が分離されている。
- Solver と Generator が分離されている。

テスト:
- `npm test` でテストが走る。
- RNG が同じ seed で同じ結果を返す。
- 一方通行床・開閉ゲートの制限をテストしている。
- Solver が最短手数を返す。
- ランキング用生成盤面が20手以上であることをテストしている。
- RankingService の保存と読み出しをテストしている。
