# HAKODASE / ハコダセ

倉庫の色分け出荷をテーマにした、ブラウザ向けの短時間タイムアタック型スライドパズルです。
色のついた箱（コンテナ）をドラッグすると、壁・他の箱・盤端に当たるまで一気に滑ります。各箱を同じ色の搬出口（盤の縁の出口ゲート）から盤外へ出すとクリア。出す順番がカギで、クリアタイムを競います。

- ビルド不要・外部依存ゼロの静的サイト（HTML / CSS / バニラ JS）
- スマートフォン（iPhone SE 相当・横幅 320px）優先、盤面は縦長 7×9
- Canvas2D 描画（将来 Three.js/WebGL に差し替え可能な設計）
- seed 付きランダム生成（逆生成で常に可解）。ランキング対象は「最短20手以上」を保証
- 「Block Out! 系の色分け搬出スライド」にインスパイアされた**独自実装**（名称・面データ・素材・文章は流用なし）

---

## 起動方法

依存パッケージのインストールは不要です。Node.js があればそのまま動きます。

```
npm run dev
```

起動したら、表示される URL（既定では http://localhost:5173/ ）をブラウザで開いてください。
※ `index.html` をファイルとして直接開くと ES Modules の制約で動きません。必ず上記サーバ経由で開いてください。

ポートを変えたいとき:

```
PORT=8080 npm run dev
```

## テスト方法

Node 標準のテストランナーだけで動きます（追加インストール不要）。

```
npm test
```

RNG の再現性、スライド/退場判定、壁での停止、ソルバーの最短手数と可解判定、生成盤面の20手保証、ランキング保存などを検証します。

## 遊び方

1. 箱をドラッグすると、その方向へ **壁・他の箱・盤端に当たるまで一気に滑ります**。
2. 箱が盤の縁に達したとき、そこに **同じ色（記号）の搬出口** があれば盤外へ出荷されます。
3. すべての箱を出荷するとクリア。**間違った順で動かすと他の箱を塞ぐ**ので、出す順番を読みましょう。
4. タイマーは最初の操作で始まり、クリアで止まります。手数は通過したマス数の合計です。
5. ドラッグ中は「滑った先（または出荷方向）」が破線/矢印でプレビュー表示されます。
6. 「やりなおし」で同じ盤面を最初から、「新しい盤面」で別の盤面を出します。seed を指定すると同じ盤面を再挑戦できます。

### 要素

- **搬出口（出口ゲート）**: 盤の縁にある色付きの出口。同じ色の箱だけが、そこから盤外へ出られます。
- **壁（障害物）**: スライドを止めます。箱どうしの塞ぎ合いと合わせて、出す順番の読みが生まれます。
- **色と記号**: 色だけでなく ●▲■◆★＋ の記号も併記しているので、色の見分けが苦手でも遊べます。

### 難易度

| 難易度 | 色数 | 盤面 | ランキング | 特徴 |
| --- | --- | --- | --- | --- |
| 練習 | 2 | 5×6 | × | 操作に慣れる小盤面 |
| 初級 | 3 | 6×8 | × | 壁が増える |
| 標準 | 4 | 7×9 | ○ | 最短20手以上を保証 |
| 上級 | 5 | 7×9 | ○ | 箱と壁が増える |
| 達人 | 6 | 7×9 | ○ | 6色の色分け出荷 |

ランキング対象（標準以上）の盤面は、最短でも20手以上かかることを生成時に保証しています。

## ディレクトリ構成

```
index.html              エントリ
styles/main.css         スタイル
src/
  main.js               統合・ゲームループ
  core/                 ゲームロジック（描画非依存・Nodeでテスト可能）
    rng.js  rules.js  solver.js  generator.js  engine.js  palette.js
  render/               描画
    renderer.js         Renderer インターフェース
    canvas-renderer.js  Canvas2D 実装
  input/pointer-input.js  Pointer Events 入力
  ui/hud.js             HUD の DOM 更新
  services/ranking.js   ランキング（localStorage / 将来オンライン化可能）
scripts/dev-server.mjs  依存なし静的サーバ
test/*.test.js          Node 標準テスト
docs/                   要件・実装計画・アーキテクチャ
```

## 設計の要点

- **ゲームロジックは描画に依存しません。** `core/` は Node だけで import・テストできます。
- 描画は `frameState`（読み取り専用スナップショット）を受け取って描くだけ。`Renderer` インターフェースを実装すれば `CanvasRenderer` を `ThreeRenderer` に差し替えられます。
- ランキングは `RankingService` として分離。将来オンライン版へ差し替え可能です。
- 「最短20手」は1スライドで通過したマス数（退場の1歩を含む）の合計で数えます。ランキング盤面は逆生成で常に可解にし、各箱→同色搬出口のマンハッタン距離総和（最短手数の下界）で20手以上を保証します。

詳細は `docs/requirements.md` / `docs/implementation-plan.md` / `docs/architecture.md` を参照してください。

## オリジナリティ

本作は「Block Out! 系の色分け搬出スライド」という遊びの骨格に着想を得つつ、テーマ・用語・絵・盤面生成・UI をすべて独自に作ったオリジナル作品です。
名称・画面構成・面データ・演出・素材・文章は既存ゲームから流用していません。画像・音声などの外部素材は一切使わず、すべて Canvas で描画しています。
